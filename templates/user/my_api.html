API Documentation
Authentication

Pass your token via the X-API-Key header or the api_key query parameter.

GET /api/v1/sms/latest?api_key=YOUR_TOKEN
— or —
Header: X-API-Key: YOUR_TOKEN
Endpoints
{# ── GET /sms/latest ── #}
GET /api/v1/sms/latest

Get the most recent SMS. Useful for bot OTP fetching.

ParamTypeDescription
servicestringFilter by service (e.g. Telegram)
phonestringFilter by phone number
sinceintOnly SMS newer than this Unix timestamp
Example response: 
{
  "ok": true,
  "sms": {
    "id": 123,
    "service": "Telegram",
    "phone": "263784137580",
    "country": "Zimbabwe",
    "message": "Telegram code 41239",
    "rate": 0.01,
    "received_at": "2026-03-05T07:13:00"
  }
}
{# ── GET /sms/list ── #}
GET /api/v1/sms/list

Get a paginated list of your SMS.

ParamTypeDescription
servicestringFilter by service
phonestringFilter by phone number
sinceintOnly SMS newer than this Unix timestamp
limitintMax results (default 20, max 100)
pageintPage number (default 1)
Example response: 
{
  "ok": true,
  "page": 1,
  "total": 42,
  "sms": [
    { "id": 123, "service": "Telegram", "phone": "263784137580", ... },
    ...
  ]
}
{# ── GET /numbers ── #}
GET /api/v1/numbers

List your allocated phone numbers.

Example response: 
{
  "ok": true,
  "total": 100,
  "numbers": [
    { "phone": "263784137580", "country": "Zimbabwe", "service": "Telegram" },
    ...
  ]
}
